1. Executive Summary

This document presents the complete technical and functional overview of the Vaswani Industries Limited corporate website and Content Management System (CMS). The platform is a unified, modern web application serving both the public-facing corporate website and the internal admin management panel on a single deployment.

Key Highlights:
  • Single unified deployment (website + admin + API on one server)
  • Real-time content management without developer intervention
  • Mobile-responsive design for all devices (iOS, Android, Desktop)
  • SEO-optimized for search engine visibility
  • Secure admin panel with JWT authentication
  • Hostinger VPS production-ready

11. Deployment Guide (Hostinger VPS)

11.1 Prerequisites

  • Hostinger VPS with Node.js 18+ support
  • MongoDB Atlas connection (cloud database — already configured)
  • Domain: www.vaswaniindustries.com pointed to VPS IP address
  • SSL certificate (via Hostinger panel)

11.2 Deployment Steps

1. Upload project files to VPS (via Git clone or SFTP)
2. Rename .env.production → .env
3. Install dependencies:
   npm install --production
4. Build the application:
   npm run build
5. Start with PM2 (process manager):
   pm2 start server.js --name vaswani
6. Configure Nginx reverse proxy:
   proxy_pass http://localhost:3001
7. Enable SSL via Hostinger panel
8. Verify: https://www.vaswaniindustries.com

12. Maintenance & Support

12.1 Regular Tasks

  • Monitor server health via /api/health endpoint
  • Check notification panel for new enquiries daily
  • Upload quarterly financial documents as per SEBI requirements
  • Update news/CSR activities regularly
  • Review and respond to contact form submissions

12.2 Content Updates (No Developer Needed)

All content updates can be done through the admin panel:

  • Edit page text, images, and buttons
  • Upload and manage investor documents
  • Publish news articles with images
  • Manage board member profiles
  • Post job openings and view applications
  • Update company contact information

12.3 Backup Strategy

  • MongoDB Atlas provides automatic daily backups (cloud)
  • Uploaded files in /public/uploads/ — backup periodically via SFTP
