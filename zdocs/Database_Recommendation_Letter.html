Subject: Database Technology Recommendation — MongoDB vs MySQL

Dear Sir,

I would like to brief you on the database choice for our website project and why switching from MongoDB to MySQL at this stage would not be advisable.

Current Setup (MongoDB)

Our website, admin panel, and backend are fully built and tested using MongoDB (cloud-hosted on MongoDB Atlas). Everything is working — all CRUD operations, file uploads, news, board members, documents, careers, contacts, and page content management.

Why MongoDB is the Right Choice for This Project

FactorMongoDB (Current)MySQL (Alternative)
Data StructureFlexible — supports nested content (page sections, image galleries, buttons) naturallyRigid — requires 15+ separate tables with complex joins
HostingMongoDB Atlas (free cloud, works with any server including Hostinger)Requires Hostinger MySQL database setup
CMS SuitabilityExcellent for content management systemsBetter for transactional/accounting systems
Development SpeedAlready complete and testedWould require 1 week+ of rewriting
CostFree tier (512MB, sufficient for our needs)Included in Hostinger plan

Risks of Switching to MySQL Now

  • Timeline Impact: Minimum 1 week additional development time to rewrite all 7 data models, relationships, and queries
  • Testing Required: Complete re-testing of all admin operations (upload, edit, delete)
  • Complexity Increase: Our page content has nested structures (sections → images → buttons). In MySQL, this requires 4-5 separate tables with foreign keys instead of 1 simple document
  • No User-Facing Benefit: The website visitor and admin user experience remains exactly the same regardless of database choice
  • Risk of New Bugs: Rewriting working code introduces potential for new issues

Estimated Timeline if We Switch

TaskDuration
Schema redesign (7 models → 15+ tables)1.5 days
ORM migration (Mongoose → Sequelize)2 days
Rewrite all API routes2 days
Testing & bug fixing1.5 days
Total~7 working days (1 week)

Recommendation

Continue with MongoDB. It is production-ready, free, cloud-hosted, and perfectly suited for our content management needs. The project can be deployed to Hostinger immediately without any database-related changes.

MySQL would be the right choice if we were building an ERP, inventory system, or financial application — but for a corporate website with CMS, MongoDB is the industry standard (used by companies like Adobe, eBay, and Forbes for similar purposes).
